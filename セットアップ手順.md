# セットアップ手順（あなたが行うこと）

AIによるVault構築は完了しています。以下を実施して動作確認してください。

## 1. ObsidianでVaultを開く

1. Obsidianを起動
2. **「フォルダをVaultとして開く」**
3. このフォルダを選択:
   ```
   C:\Users\murak\OneDrive\ドキュメント\DMZ\戦術研究所\Vault
   ```

## 2. Dataviewプラグイン（必須）

1. 設定 → **コミュニティプラグイン** → 有効化
2. **閲覧** → 「Dataview」を検索してインストール
3. Dataviewを有効化
4. `00_ダッシュボード/戦術研究ダッシュボード.md` を開き、テーブルが表示されるか確認

> テーブルがコードブロックのまま表示される場合は Dataview が未有効です。

## 3. 推奨確認項目

| 確認 | 場所 |
|---|---|
| ダッシュボード一覧 | `00_ダッシュボード/戦術研究ダッシュボード.md` |
| Canvas関連図 | `60_図解・マップ/Canvas/`（6ファイル） |
| 公開記事 | `80_Public/index.md` |
| Graph表示 | Obsidianのグラフビューで記事のつながり |

## 4. 日常の使い方

1. 実プレイ後、戦術メモを `01_Inbox` に貼る
2. `70_AI生成用素材/プロンプト/戦術メモ記事化プロンプト` をAIに渡して記事化
3. 適切なフォルダへ移動し、`status` を更新
4. 検証後 `verified` → `80_Public` へ複製

## 5. GitHub Pages（任意・後からで可）

1. Gitをインストール（現在の環境では未検出）
2. リポジトリを作成し、このVaultをpush
3. GitHub Pagesのソースを `/docs` に設定
4. `docs/index.md` がトップページになる

## 6. あなたにしかできないこと

- **実プレイに基づく記事の追記・修正**（特にコシチェイなど未確認エリア）
- **confidence の引き上げ**（実戦で再現できた戦術）
- **フレンド名・個人ログ** の `02_Private` への記録
- **画像・マップスクショ** の `Assets/images` への追加
- **公開判断**（`80_Public` へ移す最終承認）

## 7. 改善の進め方

1. Obsidianで記事を読み、誤りや不足をメモ
2. チャットで「〇〇の記事を修正して」と依頼
3. 実プレイログを `01_Inbox` に入れ、記事化を依頼
